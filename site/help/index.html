Sidekick Help

Current docs for Sidekick 0.7.0. Sidekick supports Kay and Codex sidekicks for Claude Code and Codex hosts. The host AI plans, delegates, reviews, verifies, and owns final correctness.

Fast path: activate Kay with /sidekick:kay-delegate or Codex with /sidekick:codex-delegate. Stop with /sidekick:kay-stop or /sidekick:codex-stop.
Getting StartedInstall or repair Sidekick, choose Kay or Codex, activate one sidekick, and complete the first verified task. ConceptsUnderstand the host AI, sidekick runtime, active-sidekick selector, hook boundary, and verification loop. WorkflowsTask flow through kay exec and codex exec, including task handoff, review, retry, and stop behavior. ReferenceCommands, state paths, generated host bundles, registry fields, and test runners. TroubleshootingFix activation, routing, verification, test, and release-gate failures. CompatibilityHow Claude Code, Codex, Kay, and the Codex sidekick map to the same delegation contract.

Current Contract

One active sidekick: Kay and Codex are mutually exclusive in a host session through the shared active-sidekick selector.
Host owns correctness: sidekick success text is never enough; the host checks requirements, diffs, tests, and integration behavior.
Runtime-owned execution: Kay runs through kay exec; Codex runs through codex exec using gpt-5.4-mini with extra-high reasoning.
