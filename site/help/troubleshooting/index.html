Troubleshooting

Use this page when activation, routing, verification, generated host bundles, or release checks do not match the current Sidekick contract.

Activation Fails

State

Symptom: the host says a different sidekick is already active, or delegation commands appear to route to the wrong runtime.

Fix: stop the current sidekick, then activate the one you want. Check the shared active-sidekick selector if state still looks stale.

/sidekick:kay-stop /sidekick:codex-stop cat ~/.sidekick/sessions/<session>/active-sidekick

Kay Runtime Fails

Runtime

Symptom: Kay cannot launch, provider login is missing, or the selected provider route is not accepted.

Fix: verify the runtime, check help output, then log in through Kay's provider flow. Use the session selector only after the runtime is healthy.

kay --version kay exec --help kay login --provider opencode-go --with-api-key /sidekick:kay-delegate ocg

Codex Runtime Fails

Runtime

Symptom: Codex activation succeeds but delegated work cannot start.

Fix: confirm the local OpenAI Codex CLI is on PATH and can run non-interactive tasks. Sidekick does not install Codex for you.

codex --version codex exec --help /sidekick:codex-delegate

Verification Fails

Quality

Symptom: the sidekick reports success, but tests fail, the wrong file changed, a requirement is missing, or the implementation relies on an unverified assumption.

Fix: keep the same sidekick active and relaunch it with a smaller prompt that names the failed check, the relevant file, and the desired final state. The host should verify again before reporting completion.

bash tests/run_unit.bash bash tests/run_all.bash

Generated Bundles Are Stale

Build

Symptom: Claude and Codex host surfaces show different skill text, or tests report a rendered-surface mismatch.

Fix: update the canonical skill under skills/, rerun the renderer, then run unit checks.

bash scripts/sync-host-surfaces.sh bash tests/run_unit.bash

Release Gate Fails

Release

Symptom: release checks fail in the live Codex path or the docs contract blocks the release.

Fix: run the failing test directly, update the public docs or registry contract, then rerun the Kay-hosted live release command.

bash tests/test_docs_contract.bash bash tests/test_help_site_navigation.bash bash tests/run_in_kay.bash SIDEKICK_LIVE_CODEX=1 bash tests/run_release.bash
