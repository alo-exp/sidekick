Getting Started

Start with one active sidekick per host session. Kay and Codex share the same host-owned verification contract, but they use different child runtimes.

Install or repair Sidekick. Use the host plugin surface for normal installs, or run the repository sync script when working from this checkout.
Activate one sidekick. Pick Kay for the Kay runtime or Codex for the local OpenAI Codex CLI.
Delegate a bounded task. The host turns your request into a focused child-runtime command.
Verify the result. The host inspects files, runs appropriate checks, and compares the result to the original prompt.

Install Or Refresh Host Surfaces

Generated host bundles under agents/claude/ and agents/codex/ come from canonical skill sources. Refresh them whenever skill source text changes.

/plugin install alo-labs/sidekick bash scripts/sync-host-surfaces.sh

Choose The Sidekick

Kay sidekick

Use Kay when you want Sidekick to run the Kay runtime through kay exec. Kay defaults to OpenCode Go routing, accepts xiaomi and ocg session selectors, and also respects SIDEKICK_KAY_PROVIDER.

/sidekick:kay-delegate /sidekick:kay-delegate xiaomi /sidekick:kay-delegate ocg

Codex sidekick

Use Codex when you want the local OpenAI Codex CLI to run the delegated task through codex exec. Sidekick routes this mode to gpt-5.4-mini with extra-high reasoning.

/sidekick:codex-delegate

Stop Or Switch

Stop the active sidekick before returning to direct host implementation or activating the other sidekick.

/sidekick:kay-stop /sidekick:codex-stop

First Verification Pass

Ask the host to run the narrowest useful checks. For this repo, start with unit checks, then expand to the full skip-safe suite when the change crosses site, hook, installer, or packaging boundaries.

bash tests/run_unit.bash bash tests/run_all.bash
