SECTION: # /forge -- Forge Delegation Mode

Brief intro: This skill adds explicit activation/deactivation mode switching on top of the existing `skills/forge.md` orchestration protocol. It does NOT replace forge.md -- it wraps it with a persistent session state mechanism.

SECTION: ## Activation (`/forge`)

Step-by-step protocol:

1. Health Check -- All 4 must pass:
   - `~/.local/bin/forge` exists OR `which forge` succeeds
   - `forge info` exits 0 and output contains a provider name (non-empty line after "Provider:")
   - `~/forge/.credentials.json` exists and contains non-empty `api_key` field
   - `~/forge/.forge.toml` exists and contains non-empty `provider_id` and `model_id`
   - If ANY check fails: Print which check failed and reference `skills/forge.md` STEP 0A for setup instructions. Do NOT duplicate the install flow. Stop activation.

2. Bootstrap Config (first invocation only, per D-05 non-destructive rule):
   - If `.forge/agents/forge.md` does not exist: create it (content defined in plan 01-03)
   - If `.forge.toml` does not exist in project root: create it (content defined in plan 01-03)
   - If `.forge/skills/` directory does not exist: create it and populate bootstrap skill set (content defined in plan 01-03)
   - If any of these already exist: skip silently, never overwrite

3. Set Session State:
   - Create zero-byte file: `~/.claude/.forge-delegation-active`
   - If file already exists (stale from prior session): re-validate health check, then acknowledge "Forge-first mode is already active (re-validated)."

4. Confirm to User:
   - "Forge-first delegation mode activated. All implementation tasks will be routed to Forge."
   - "To deactivate: `/forge:deactivate`"

SECTION: ## Delegation Protocol (while active)

- Before every task, check: `[ -f ~/.claude/.forge-delegation-active ]`
- If active: follow `skills/forge.md` STEP 1 through STEP 9 for task execution
- DLGT-04 enforcement: While marker exists, Claude MUST NOT directly use Write, Edit, or Bash tools for implementation work. Exception: Level 3 fallback (Phase 2).
- Claude composes task prompts per the format in the spec (section 4): OBJECTIVE, CONTEXT, DESIRED STATE, SUCCESS CRITERIA, INJECTED SKILLS
- Claude communicates progress and outcomes to user in plain language (DLGT-05)

SECTION: ## Deactivation (`/forge:deactivate`)

1. Check if `~/.claude/.forge-delegation-active` exists
   - If yes: delete it, confirm "Forge-first mode deactivated. Claude-direct mode restored."
   - If no: acknowledge "Forge-first mode is not currently active."
