## practice:plugin-packaging

- 2026-05-07 — **Split command content from discovery plumbing.** Keep the actual workflow in one canonical Markdown file, then add a thin bridge file that only points back to it. That pattern preserves a single source of truth while still letting runtimes expose the command in their native picker or import path.

## practice:documentation

- 2026-05-07 — **Write packaging guides around the runtime's real discovery mechanism, not the idealized manifest.** If the runtime surfaces workflows through imported skills or bridge files, document that behavior directly so other plugins can reproduce the result instead of duplicating the content.
- 2026-05-08 — **Keep the canonical instruction body in the universal skill format and let commands point to it.** That keeps one source of truth while still giving slash-command runtimes a familiar entrypoint.
