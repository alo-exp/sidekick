## Architecture Patterns

- 2026-04-18 — **Forge-first delegation is harness-enforced, not prompt-suggested.** v1.2 added a PreToolUse hook (`hooks/forge-delegation-enforcer.sh`) that denies `Write`/`Edit`/`NotebookEdit` while the marker file exists, and rewrites `Bash forge -p …` to inject a valid UUID `--conversation-id` + `--verbose` + `[FORGE]`/`[FORGE-LOG]` line prefixes. Prompt-only enforcement was insufficient — Claude could rationalize around it; harness layer cannot be bypassed.
- 2026-04-18 — **Sidekick indexes, Forge stores.** `.forge/conversations.idx` is a lookup-only audit row (ISO-UTC + UUID + sidekick-tag + task-hint). All content, stats, and replay come from Forge's native `~/forge/.forge.db` via `forge conversation dump/stats/info`. This avoids reinventing storage and keeps Sidekick upgrade-compatible with Forge.

## Known Gotchas

- 2026-04-18 — **`--conversation-id` must be a lowercase RFC 4122 UUID.** Forge 2.11.3 rejects the earlier `sidekick-<ts>-<hash>` format. The human-readable tag is preserved as a separate column in `.forge/conversations.idx` for display only; the UUID is what gets passed to Forge.
- 2026-04-18 — **Claude Code PreToolUse hook JSON shape** uses `hookSpecificOutput.{hookEventName, permissionDecision, permissionDecisionReason, updatedInput.command}` — not the `decision`/`modifiedCommand` shape drafted in the v1.2 spec.
- 2026-04-18 — **Output styles do NOT style tool output by line prefix.** They shape Claude's narration prose. v1.2 ships `output-styles/forge.md` as a narration contract documenting `[FORGE]`/`[FORGE-LOG]`/`[FORGE-SUMMARY]` markers that Claude is expected to preserve when relaying — not as a color renderer for raw subprocess output.
- 2026-04-18 — **Classifier gap: `sed -i` and `awk -i inplace` slip past naive read-only matching.** The first-word match on `sed`/`awk` treated them as read-only until an explicit reject was added ahead of the fallback. Covered by `tests/test_v12_coverage.bash`.

## Key Decisions

- 2026-04-18 — **Six-tier pre-release test pyramid** (`tests/run_release.bash`): unit + integration (`run_all.bash`, always runs) → Forge smoke (`forge --version` + trivial `forge -p`, gated on `SIDEKICK_LIVE_FORGE=1`) → Forge live E2E (seeded-buggy testapp round-trip, same gate) → Kay marketplace install (`run_live_codex_marketplace_install.bash`, gated on `SIDEKICK_LIVE_CODEX=1`) → Kay smoke (`kay --version` + trivial `kay exec`, same gate) → Kay live E2E (seeded-buggy testapp round-trip, same gate). CI runs stage 1 only; maintainer runs all six with the env vars before tagging.
- 2026-04-18 — **Forge agent template MUST declare `tools: ["*"]`.** Absence silently provisions zero tools; the model emits XML-looking text that never executes, and Forge reports `STATUS: SUCCESS` with nothing actually done. Fixed in v1.1.2.
